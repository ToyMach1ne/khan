Installation top

Installing the theme can be done two ways. You can upload the theme ZIP file using the built in WordPress theme upload, or copy the files through an FTP client.

1) Using WordPress Theme Upload

Be sure to extract the file "theme-vellum.zip" from the ThemeForest download before uploading. Using the ThemeForest ZIP file directly will result in a "Missing Style Sheet" error.

  1. Login to your WordPress admin.
  2. In the "Appearance > Themes" menu click the tab "Install Themes"
  3. At the top of the page click, "Upload", then click the file input to select a file.
  4. Select the zipped theme file, "theme-vellum.zip", and click "Install Now"
  5. After installation you will receive a success message confirming your new install.
  6. Click the link "Activate"
2) Uploading by FTP
  1. Login to your FTP server and navigate to your WordPress themes directory. 
    Normally this would be "wp-content/themes"
  2. Extract the files from the zipped theme.
  3. Copy the folder "parallelus-vellum" to your themes directory.
  4. After the files finish uploading, login to your WordPress admin.
  5. In the "Appearance" menu click "Themes"
  6. Click "Activate" for the theme "Vellum"
Activate the Theme

Once the theme is installed, activate it from "Appearance > Themes" by clicking the "Activate" link for "Vellum".

Demo Data and Starter Kits (Optional)

You can import any demo styles you choose from the "Appearance > Demo Content" area by selecting one of the Starter Kits and sleecting "Apply with Demo Content" which will set the theme options, layouts and other theme specific settings for that style as well as import the demo content and menus at the same time. If you've already imported the demo content and only want to activate a different Starter Kit, click the "Apply" button.

If you prefer the full demo content file, it is available by request. To get a copy, please go to our Help Center where we can provide access to this file.

Setting the Home Page:

After installing the demo content you may need to specify the home page content in your WordPress settings. Go to "Settings > Reading" and set the options for "Front page displays" to "A static page". Next in the "Front page" select choose "Home Page" and in the "Posts page" select "Blog". Once this is done your site should replicate the demo.

Assigning a Menu:

The demo content also includes several menus. The theme need a primary menu to be assigned from the "Appearance > Menus" screen of your WordPress admin. Go to Menus area and select the option "Main Menu" from the area labeled "Select a menu to edit" and click the "Select" button. Next scroll to the bottom of the menu and in the "Menu Settings" select the option for "Theme locations" labeled as "Primary Menu" and save the menu.

To enable the mega menus you should expand the individual top level menu items, expand the button "Show/Hide UberMenu Options" and select the option "Activate Mega Menu". Do this for each of the top level menu items you want to have a mega menu display and save the menu.

Now the site will display this is done your site should replicate the demo.

Appearance top

This section provides a brief overview of the theme options, setup and configuration. It will help you get started quickly by introducing the theme structure and settings. After activation several new menus will appear in your admin under the sections, "Appearance" and "Settings".

  1. Logo

You will find a number of other settings available in this area including skin selection, fonts, custom CSS and custom JavaScript. Use these as needed to customize the look and feel of your site.

Sidebars (Appearance > Sidebars)

Create custom sidebars for adding widgets, assign to layouts and inserting with shortcodes.

New sidebars instantly appear in the "Widgets" area of your admin. To integrate a sidebar into one of your pages you can assign it to a layout from "Appearance > Layout Manager" or use the shortcode, [sidebar alias="sidebar-alias"]

Layout Manager (Appearance > Layout Manager)

Build custom layouts. Each layout can have a custom Header, Footer, Skin and Layout style (boxed or full width). Layouts can be assigned to your pages, posts, categores and other site areas.

Every layout should have one "Default Content" area. This sets the location of the main content. If not included it will be added to the end of your layout.

The columns in your layouts can be dragged left and right to create different width columns.

Layouts are designed to create the default structure of your site. They specify the source of content such as sidebars and Static Blocks. These are structural forms to designate the column structure, they are not intended for building the page content. That should be done from the page, post, etc. We've integrated the WP Bakery Visual Composer plugin into the theme for content building functionality. Think of the Layout Manager as the the structural framework for your site's content and Visual Composer as your content editing tool.

UberMenu Lite (Appearance > UberMenu)

Select your default options for the mega-menu functionality. Individual menu item settings can be done from the WordPress menu manager under the options for each item.

What's UberMenu Lite?

The theme includes UberMenu Lite, a custom verson of UberMenu with core mega-menu functionality including titles, descriptions, columns, rows and icons using Font Awesome class names. We worked directly with SevenSpark, the creator of UberMenu, to develop UberMenu Lite and include it in our themes. The Lite version provides the theme specific mega menu features.

Should I ugprade to the full Pro version?

You can install the Pro version for advanced features such as Images, Widgets, Custom HTML Content, Google Maps, and much more. The theme is designed to instantly integrate with Pro installs. If you already own a copy of the UberMenu plugin, simply update to the latest version and you're ready to go.

If you don't already have UberMenu Pro but want some of those awesome features you can get your copy here: UberMenu - WordPress Mega Menu Plugin.

Blog Options (Settings > Blog Options)

Configure some of the blog display options found in "Settings > Blog Options". These options control the display of content on your blog post lists and single post pages.

Blog Post List

  • Default Template - The default blog display style.
  • Default Grid Columns - If a grid layout is selected as the default, this allows you to set the number of columns.
  • Use post excerpts - Show content excerpts on your blog pages. Unchecking this will display the full post in your post list.
  • Excerpt length - The number of words in post excerpts, 250 max. Custom excerpts are not restricted by this setting.
  • Read More Link - Optional link after post excerpts. Enter desired text for the link or for no link, leave blank or set to "-1".
  • Image Width - The maximum post image width. Images will be scaled using this as a guide.
  • Image Height - The maximum post image height. Images will be scaled using this as a guide.
  • Show Post Format - Show or hide the post format icon in the blog list.
  • Show Date - Show or hide the post date in the blog list.
  • Show Categories - Show or hide the category list in the blog list.
  • Show Comment Count - Show or hide the comment count in the blog list.
  • Show Tags - Show or hide the tags list in the blog list.

Single Post

  • Show Featured Image - Show image for current post on the single post page. Unchecking this will show post content only.
  • Image Width - The maximum post image width. Images will be scaled using this as a guide.
  • Image Height - The maximum post image height. Images will be scaled using this as a guide.
  • Show Post Format - Show or hide the post format icon in the single post.
  • Show Date - Show or hide the post date in the single post.
  • Show Categories - Show or hide the category list in the single post.
  • Show Comment Link - Show or hide the comment count in the single post.
  • Show Post Navigation - Show or hide the next/previous post link options in the single post.

Post Formats

Set the text for each blog Post Format as you like it to be displayed on the public website.

  • Audio Format
  • Gallery Format

Blog pages can be specified by selecting one of the blog templates under the Pages Attributes > Template options or using a [blog] shortcode.

When selecting a Template you can choose from multiple layouts. The settings for the post list will use the defaults specified in "Settings > Blog Options". If using a [blog] the settings can be specified for each instance to override the defaults.

You should NOT set a Template when using the [blog] shortcode.

Blogs from Shortcodes

The [blog] shortcode allows you to add a blog post list to any area of your site. The shortcode can have any parameters available in the WP_Query function. See a full list here: WP_Query Parameters.

Array variables should be separated by commas: "value_1,value_2"
For example, [blog tag_slug__in="break,baking"] will query posts with tags "bread" or "baking".

The [blog] shortcode also accepts custom parameters:

  • template
    Specify the template name (with .php) for the layout. This accepts "blog-image-left" or "blog-image-top" (default).

    Example: [blog template="blog-image-left"]

  • paging
    Show the paging: « 1 2 » Values: true (default), false

    Example: [blog paging="false"]

Contact Forms (Settings > Contact Forms)

Display your form with the [contact_form] shortcode. You can customize each instance with unique values for "to", "subject" and "thankyou" in the form. Empty fields use the default settings established in the fields below.

  • Email To - The default address to deliver messages sent from contact forms.
  • Subject - Enter the default email subject for contact form messages.
  • Thank You Message - The "thank you" message visitors will see after sending.
  • Button text - The text to appear on the send button. Default: "Send"
  • Use CAPTCHA - Require CAPTCHA image verification?

Contact Form Fields

You can create new fields to add to your contact form.

  • Field title - The name of the field as it will be displayed in the form.
  • Field key - This key is used to add the field to contact forms.
  • Caption - You can add instructions or information for the user about this field. HTML code is allowed.
  • Field type
    • Text - A simple one line text input.
    • Textarea - Plain text box for multiple lines of text.
    • Select - A select list (drop down), using the names/values specified in the "Values" field below.
    • Radio button (set) - A radio button set, where the user can select a single option. The list of values is specified below.
    • Checkbox - Single checkbox.
    • Hidden - A hidden field not seen by the user. Specify the value in the field below.
  • Values - Set the value of hidden fields here or they will contain no information. If your selected field type requires pre-defined options, such as radio buttons or select boxes, enter the values here as a comma separated list. For example, your values could be entered as Good, Better, Best.
  • Required - Require users to enter a value?
  • Required error message - Enter an optional error message for empty fields that are required.
  • Minimum length - You can optionally specify a minimum number of characters allowed for this field.
  • Maximum length - You can optionally specify a maximum number of characters allowed for this field.
  • Validation - You can apply validation to some fields to ensure valid entries.
  • Validation error message - Enter an optional error message for fields that fail validation.
  • Input width - Optional. You can specify the width of the field in pixels. Does not apply to some input types.
  • Input height - Optional. You can specify the height of the field in pixels. Does not apply to some input types.

Portfolios top

The theme includes a custom post type for creating portfolio items. To add new portfolio items go to the "Portfolio > Add New" option in your admin menu.

You can specify the details for your portfolio item add it to categories which can be used to create a filtered portfolio page or list items from a specific category only.

Portfolio pages can be specified from a standard WordPress page by selecting a portfolio template from the Pages Attributes > Template options or using a [portfolio] shortcode. This will make a standard WP page display as a custom portfolio.

When selecting a Template you can choose from multiple layouts.

You should NOT set a Template when using the [portfolio] shortcode.

Portfolios from Shortcodes

The [portfolio] shortcode allows you to add a portfolio to any area of your site. The shortcode can have any parameters available in the WP_Query function. See a full list here: WP_Query Parameters.

Array variables should be separated with commas: "value_1,value_2"
For example, [portfolio post__in="12,9"] will query items with ID 12 and 9.

The [portfolio] shortcode also accepts custom parameters:

  • template
    Specify the template name (with .php) for the layout. This accepts "grid-rows", "grid-rows-filtered", "grid-staggered" and "grid-staggered-filtered".

    Example: [portfolio template="grid-rows-filtered"]

  • paging
    Show the paging: « 1 2 » Values: true (default), false

    Example: [portfolio paging="false"]

  • columns
    Specify a number of columns (default: 3).

    Example: [portfolio columns="5"]

  • taxonomy_slug and taxonomy_terms
    While not specific to the portfolio, these parameters are for custom taxonomies. They must BOTH be entered into the shortcode. This simplifies custom taxonomy queries.

    Example:
    [portfolio taxonomy_slug="portfolio-category" taxonomy_terms="print,music,video"]

Static Blocks top

There is a custom post type called "Static Content" used to make reusable content areas for the site. After installing the theme you can import the demo content wich includes the default static blocks used for creating header, footer and sidebar content.

Using Static Content Blocks

Static block content can be inserted into your Headers, Footers, Layouts and directly into other content using the shortcode: [static_content id="id-or-slug"]

When inserting a static content block with the shortcode, the ID should be the slug for that block. The slug is found just below the title field when editing and appears as the end of the URL path to the page: "mysite/featured-products/"

Static Block Widgets

Static blocks may also be inserted using the Static Block widget. Drag the widget into any sidebar and select the desired block to use. You may also choose to optionally show the title.

 

Theme Code Structure top

The theme uses admin layouts to format your template files. You can specify additioal template files, post types and taxonomies to be selectable for custom layout options in the Layout Manager.

Template Files and Post Types

In this template structure it is possible to include more files into the options of the Layout Manager by registering them with the theme. Most files like Custom Post Types are registered automatically by the theme. In some cases the theme is not able to register a file automatically, although it tries to automate this process as much as possible. When a new or custom template file needs to be registered you can add a reference to the "functions.php" file as showin in the example below:

# An example of registering a custom layout to a template file
# -------------------------------------------------------------

$sample_file = locate_template('taxonomy-sample.php'); 
register_context( 'Sample', 'sample_tax', $sample_file);

This information is replicated in greater detail in the theme admin Layout Manager area. If you need help or have questions, please visit our Help Center.

CSS Files and Structure top

There are several CSS files included with this theme, but only a few will be necessary for customization. The CSS files are all located in the root theme folder. The only exception to this is plugin specific style sheets which are located in a sub-folder named "css". Within that folder there is a sub-folder, "source".

The sub-folder, "css/source", contains the original CSS files with formatted code and comments. Only a some files are in this folder. These files might include the original CSS files for add-on scripts used in the design. If you find a CSS file that is uncommented and you need the original, check this folder.

Skin File Naming

The skin CSS files are also formatted following a similar naming structure as templates and design files, however the file name is less important and does not dictate functionality for skins. These files are named with the prefix "style-skin", for example "style-skin-1.css".

The theme recognizes new skin CSS files automatically by adding the following comment to the beginning of the file:

/*
	Skin Name: Skin 1
*/

The name you enter here is what will appear in the admin skin menu. Once you upload your new skin CSS file you will see the new skin automatically in your skins drop down menu.

Skinning the Theme top

Each "skin" is controlled by a custom CSS file. These are located in the theme folder "parallelus-vellum". If you need to create custom graphics for you skin this can be done easily by using one of the files located in the "Design" folder included with the theme.

You can create a custom skin in a few minutes by duplicating an existing skin CSS file and editing the parameters. This can be a useful way to make structural changes to the theme design without modifying the core CSS files or theme templates.

To create a single skin for your site

The best way to go about creating your own skin, is by starting from one of the pre-made skins included with the theme. An easy way to do this would be as follows:

  1. First, create a copy of an existing style sheet from the theme folder "style-skin-#.css", and increment the # to the next unused value. Update the skin name in the comments at the beginning of the file as well.
  2. Open the new CSS file for your skin. Edit the styles or add your own custom styles.
  3. Upload your new skin files to the theme folder "parallelus-vellum" on your server.
  4. Select the new skin from a page or your default theme options "Appearance > Theme Options.
  5. Save your settings and preview the new skin.

Popup Links

The lightbox included with the theme can easily be used to load content from other pages, websites, videos and images. Many of these features are covered by the shortcodes for portfolio and styled images, but you can also manually add these features.

Load a web page using an iframe

You can create links to other pages or sites and open them in an iframe two ways.

  • Add the class "iframe":
    <a href="http://www.google.com" class="iframe">Google</a>
  • Add "#iframe" to the URL:
    <a href="http://www.google.com#iframe">Google</a>
Load an image or video

You can create links to images and videos in two ways.

  • Add the class "popup":
    <a href="myImage.jpg" class="popup">View My Image</a>
  • Add "#popup" to the URL:
    <a href="myImage.jpg#popup">View My Image</a>
Login popup

Launch the login popup by setting a link URL to #LoginPopup and assigning the class popup.

  • Standard link:
    <a href="#LoginPopup" class="popup">Sign in</a>

  • Add to the WordPress menu:
    Inclucing in the WordPress menu requires the same, the URL set to #LoginPopup and assigning the class popup
